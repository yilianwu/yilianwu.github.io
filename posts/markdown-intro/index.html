Intro to Markdown

2019-02-03

最近發現用Markdown來做筆記超好用的！ 如果是平常用pages、word或記事本，常會因為寫完沒好好整理導致忘了什麼東西寫在哪，又或是整理起來很花時間，到最後又懶得整理了XDD

Markdown常見於Github的readme.md檔 他的語法不會像HTML那樣繁雜，相反的他可以用很簡單很簡單的語法就能表現各個樣式的文字～

Markdown 常用語法

• 標題

# 一級標題
## 二級標題
### 三級標題
#### 四級標題
##### 五級標題
###### 六級標題

一級標題

二級標題

三級標題

四級標題

五級標題
六級標題

• 列表

- 無順序列表
- 無順序列表
- 無順序列表

或是

* 無順序列表
* 無順序列表
* 無順序列表

又或是

+ 無順序列表
+ 無順序列表
+ 無順序列表
  • 無順序列表
  • 無順序列表
  • 無順序列表
1. 有順序列表
1. 有順序列表
1. 有順序列表
  1. 有順序列表
  2. 有順序列表
  3. 有順序列表

• 可勾選式清單

- [x] @mentions, #refs, [links](), **formatting**, and <del>tags</del> supported
- [x] list syntax required (any unordered or ordered list supported)
- [x] this is a complete item
- [ ] this is an incomplete item

• 引用文字

> Four has four letters

Four has four letters

• Hightlight 字元

  *我是斜體*
  _我也是斜體_

  **我是粗體**
  __我也是粗體__

  ***我是粗斜體***
  ___我也是斜粗體___

  ~~我被刪除~~

我是斜體 我也是斜體

我是粗體 我也是粗體

我是粗斜體 我也是斜粗體

我被刪除

• 插入圖片

![幫圖片命名](圖片的連結)

• 插入連結

[欲顯示的超連結文字](欲前往的連結)
[我的Blog首頁](https://yilianwu.github.io)

我的Blog首頁

• 插入程式碼

  • 單行程式碼

    ` 我是單行程式碼 `

    我是單行程式碼

  • 區塊程式碼

    ` ` `
多行程式碼包在這裡面 #把` `之間的空格刪除
` ` `

    多行程式碼包在這裡面

• 表格

First Header | Second Header
------------ | -------------
Content from cell 1 | Content from cell 2
Content in the first column | Content in the second column
First Header Second Header
Content from cell 1 Content from cell 2
Content in the first column Content in the second column
